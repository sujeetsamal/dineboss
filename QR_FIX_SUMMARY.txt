# QR Public Access Fix - Quick Summary

## The Problem ❌

**Issue**: QR menu scanning fails with "insufficient permissions" on external networks

**Why**: Firestore security rules required **authentication** for everything
- Same network: Worked (debug/emulator mode)
- External network: Failed ("Missing or insufficient permissions")

```
Customer on 4G Network:
1. Scan QR code → /qr/restaurant-123/table-5
2. Page loads /qr page successfully
3. Try to fetch menu (subscribeToMenu)
4. Firestore checks: "isStaff(rId)" → NO (customer not authenticated)
5. ❌ ERROR: Missing or insufficient permissions
6. Blank menu screen - customer can't order
```

---

## The Solution ✅

**What was changed**: `firestore.rules` (ONE FILE)

### Before ❌
```javascript
// Customer blocked from reading restaurant data
match /restaurants/{rId} {
  allow read: if isStaff(rId);  // Only authenticated staff can read!
}

// Customer blocked from reading menu
match /restaurants/{rId}/menu/{itemId} {
  allow read: if isStaff(rId);  // Only authenticated staff!
}

// Customer blocked from creating orders
match /orders/{rId}/orders/{orderId} {
  allow create: if isAuth() && isStaff(rId);  // Must be authenticated!
}
```

### After ✅
```javascript
// Customers can see restaurant info
match /restaurants/{rId} {
  allow read: if true;  // Anyone can read
  allow create: if isAuth();
  allow update, delete: if isAdmin(rId);  // Admins only
}

// Customers can see menu
match /restaurants/{rId}/menu/{itemId} {
  allow read: if true;  // Anyone can read
  allow write: if isAdmin(rId);  // Admins only
}

// Customers can place orders
match /orders/{rId}/orders/{orderId} {
  allow create: if true;  // Anyone can create
  allow read: if isStaff(rId);  // Staff only see existing
  allow update: if isStaff(rId);  // Staff only update
  allow delete: if isAdmin(rId);  // Admins only delete
}
```
